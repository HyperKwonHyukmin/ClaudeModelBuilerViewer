설계 이유

순환 참조를 금지하는 이유: 변경의 영향 범위를 예측 가능하게 하기 위해서입니다. Core를 수정하면 모든 프로젝트가 영향받지만, Cli를 수정해도 Core는 영향받지 않습니다.

각 프로젝트 역할:

  Cmb.Core — 순수 도메인. 외부 NuGet 전혀 없음. 가장 안정적.

  Cmb.Io — 입출력. CSV 읽기, BDF/JSON 쓰기.

  Cmb.Pipeline — 알고리즘. 8개 파이프라인 스테이지, 공간 자료구조.

  Cmb.Cli — 진입점. 사용자 인터페이스(커맨드라인).

대체 방식

ExtraData: Dictionary<string,string> → EntityCategory / NodeTags 로 대체.
이유: 딕셔너리는 어떤 값이 들어올지 컴파일 타임에 알 수 없어 타입 안전성이 없음. 열거형으로 표현하면 코드 분석 도구가 모든 경우를 추적 가능.

Console.WriteLine 로깅 → ILogger 사용.
이유: 테스트에서 로그를 캡처하거나, 프로덕션에서 파일/구조화 로그로 쉽게 전환 가능.

하드코딩 tolerance → RunOptions.Tolerances 레코드에만 정의.
이유: 값이 한 곳에만 있어야 실수로 일부만 변경하는 문제를 방지할 수 있음.
