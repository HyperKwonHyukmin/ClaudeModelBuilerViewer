Phase C — cmb build-full 파이프라인 흐름
파이프라인 실행 방식

PipelineRunner가 8개 Stage를 순서대로 실행. 각 Stage는 StageContext(FeModel + RunOptions + Diagnostics)를 받아 in-place 변환 후 StageResult를 반환.

--stopat <StageName> 옵션: 특정 Stage 직후에 중단하고 그 시점의 FeModel을 JSON으로 저장.

순서Stage주요 변환출력 파일
01 SanityPreprocess 중복/단축 요소 제거, 공선 노드 병합 01_SanityPreprocess.json
02 Meshing 긴 빔 등분할 (Structure≤2000mm, Pipe≤1000mm) 02_Meshing.json
03 NodeEquivalence 근접 노드(≤1mm) 병합, 루프 요소 제거 03_NodeEquivalence.json
04 Intersection 교차 빔에 노드 삽입 + 2분할 (수렴 루프) 04_Intersection.json
05 WeldNode IsWeld 힌트 → NodeTags.Weld 태그 부여 05_WeldNode.json
06 GroupConnect 분리 그룹 연결, 고립 그룹 경고 06_GroupConnect.json
07 UboltRbe U-bolt snap → RBE2 생성 (2-phase) 07_UboltRbe.json
08 FinalValidation 무결성 검증 + BDF 출력 08_FinalValidation.json + final.bdf
핵심 데이터 객체
RawDesignData (Cmb.Io)
CSV 파싱 결과 컨테이너. 3종 DTO 목록 + 파싱 실패 행(ParseSkip) 수집.
IReadOnlyList로 불변성 보장 — Builder가 실수로 수정 불가.
Phase A 출력의 직렬화 대상.
FeModel (Cmb.Core)
모든 FEM 엔티티의 최상위 컨테이너. Phase B부터 Pipeline 끝까지 이 객체 하나가 누적 변환됨.
LengthUnit = "mm" 고정. IdAllocator로 전역 유일 ID 관리.
Nastran BDF 파일 1개에 1:1 대응.
StageContext (Cmb.Pipeline)
파이프라인 실행 시 각 Stage에 전달되는 실행 컨텍스트.
FeModel + RunOptions.Tolerances + DiagnosticList 포함.
Tolerances는 여기에서만 읽음 — 코드 내 하드코딩 금지 원칙.
JSON meta 필드
모든 출력 JSON에 공통으로 첨부:
phase: "A" / "B" / "C"
stageName: "parse" / "initial" / "NodeEquivalence" 등
unit: "mm" (고정)
schemaVersion: "1.0"
