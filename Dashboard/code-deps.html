의존성 그래프

핵심 원칙: 의존성은 반드시 단방향이어야 합니다. Cmb.Cli → Cmb.Pipeline → Cmb.Io → Cmb.Core 방향으로만 흐릅니다.
역방향 참조 또는 순환 참조는 빌드 오류로 차단됩니다. Cmb.Core는 외부 NuGet 없이 BCL(System.Text.Json)만 허용.

NuGet 패키지 의존성 정책
프로젝트외부 NuGet허용 이유 / 제한 이유
Cmb.Core 없음 (BCL만) AOT 호환, 최소 의존성, 가장 안정적인 레이어여야 함
Cmb.Io 필요시 CSV 파서 라이브러리 허용 현재 직접 구현. 복잡한 CSV 포맷 지원 필요 시 추가 검토
Cmb.Pipeline MathNet.Numerics (SVD 필요 시) 실제 필요 시점에만 추가. Directory.Build.props에 버전 고정
Cmb.Cli System.CommandLine CLI 파싱 전용. --help/--version 자동 생성, 타입 안전 옵션 파싱
*.Tests xUnit, FluentAssertions, Verify.Xunit 테스트 전용. 골든 스냅샷(Verify), 가독성 좋은 어서션(FluentAssertions)
