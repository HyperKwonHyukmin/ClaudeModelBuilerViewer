1. Sweep-and-Prune — 노드 동일성 판별 (NodeEquivalence)
참조: Pipeline/NodeInspector/InspectEquivalenceNodes.cs
목적
HiTess CSV 파싱 시 같은 물리적 위치를 가리키는 노드가 미세한 좌표 차이(부동소수점 오차, 단위 변환 오차)로 인해 다른 노드로 등록됩니다. 이 노드들을 찾아 하나로 병합합니다.
알고리즘 단계
1. X축 정렬: 모든 노드를 X 좌표 기준으로 오름차순 정렬 — O(N log N)
2. 슬라이딩 윈도우: 왼쪽 포인터를 이동하며, 현재 노드와의 X 좌표 차이가 NodeMergeTol(1mm) 초과이면 윈도우 왼쪽 경계를 전진
3. 3D 거리 확인: 윈도우 내 노드 쌍끼리 실제 3D 유클리드 거리 계산. 1mm 이내이면 같은 그룹
4. UnionFind 병합: 같은 그룹의 노드들을 UnionFind 자료구조로 묶음
5. 대표 노드 선정: 각 그룹에서 가장 작은 ID를 canonical(대표) 노드로 선정
6. 참조 교체: BeamElement.StartNodeId, EndNodeId, RigidElement의 모든 NodeId를 canonical ID로 교체
7. 루프 요소 제거: 교체 후 StartNodeId == EndNodeId가 된 요소(길이 0 요소) 삭제
복잡도
정렬: O(N log N) / 윈도우 스캔: O(N × W) where W = 평균 윈도우 크기 (보통 소수) / UnionFind: O(α(N)) ≈ O(1)
SpatialCellSizeMm(200mm)을 NodeMergeTol(1mm)보다 훨씬 크게 유지해야 합니다. 셀 크기가 너무 작으면 경계 노드가 탐지 안 될 수 있습니다.
// 의사 코드 nodes.Sort(n => n.Position.X); var uf = new UnionFind(nodes.Count); int left = 0; for (int right = 0; right < nodes.Count; right++) { while (nodes[right].X - nodes[left].X > NodeMergeTol) left++; for (int k = left; k < right; k++) { if (nodes[right].DistanceTo(nodes[k]) <= NodeMergeTol) uf.Union(right, k); } } // canonical ID로 모든 참조 교체 foreach (var elem in model.Elements) elem.StartNodeId = uf.Find(elem.StartNodeId); elem.EndNodeId = uf.Find(elem.EndNodeId);
2. Dan Sunday Segment-Segment — 교차점 분할 (Intersection)
참조: Pipeline/ElementModifier/ElementIntersectionSplitModifier.cs
목적
3D 공간에서 두 선분(빔 요소)이 교차하는지 판별하고, 교차하면 교차점에 새 노드를 삽입해 두 빔 각각을 2분할합니다. 이를 통해 "교차하지만 연결 안 된" 구조를 "교차점에서 연결된" 구조로 수정합니다.
핵심: 두 선분 사이 최단 거리
두 3D 선분 P(s) = P0 + s·(P1-P0), Q(t) = Q0 + t·(Q1-Q0)에 대해:
w = P0 - Q0, u = P1-P0, v = Q1-Q0
a = u·u, b = u·v, c = v·v, d = u·w, e = v·w
denom = a·c - b·b
s* = (b·e - c·d) / denom
t* = (a·e - b·d) / denom
최근접점: P(s*), Q(t*) / 최단 거리: |P(s*) - Q(t*)|
알고리즘 단계
1. 후보 쌍 추출: ElementSpatialHash로 서로 가까운 요소 쌍만 추출 (전체 O(N²) → 근접 쌍만)
2. 거리 계산: Dan Sunday 알고리즘으로 두 선분 사이 최단 거리 및 s*, t* 파라미터 계산
3. 교차 판정: 거리 < IntersectionTol(0.1mm) AND s* ∈ [0,1] AND t* ∈ [0,1]이면 교차
4. 노드 삽입: 교차점 좌표 = P(s*)와 Q(t*)의 중점. 새 노드 ID 발급 후 삽입
5. 요소 분할: 원래 빔 A → [A_start, 교차점], [교차점, A_end]. 빔 B도 동일하게 2분할
6. 수렴 반복: 분할 후 새 교차가 생길 수 있으므로 변화 없을 때까지 반복 (최대 MaxConvergenceIterations=50회)
평행 선분(denom ≈ 0)은 교차 판정에서 제외합니다. 수렴하지 않으면 CONVERGENCE_LIMIT_REACHED 진단을 발행하고 중단합니다.
// 의사 코드 bool changed = true; int iter = 0; while (changed && iter++ < MaxConvergenceIterations) { changed = false; var candidates = spatialHash.GetNearbyPairs(); foreach (var (a, b) in candidates) { var (dist, sA, tB) = DanSunday(a.Segment, b.Segment); if (dist < IntersectionTol && InRange(sA) && InRange(tB)) { var pt = Midpoint(a.PointAt(sA), b.PointAt(tB)); var newNode = model.AddNode(pt); Split(a, newNode); // a → a1 + a2 Split(b, newNode); // b → b1 + b2 changed = true; } } }
3. Union-Find (경로 압축 + 랭크 병합)
참조: Pipeline/Utils/UnionFind.cs
사용 Stage
NodeEquivalence — 근접 노드 그룹화 / GroupConnect — BeamElement 연결 그룹 식별
경로 압축 (Path Compression)
Find(x) 호출 시 x에서 루트까지의 경로상 모든 노드를 직접 루트에 연결.
다음 Find() 호출은 즉시 루트를 반환 — 재귀 깊이를 1로 줄임.
랭크 병합 (Union by Rank)
두 트리를 합칠 때 높이가 낮은 트리를 높은 트리 아래에 붙임.
트리 높이가 O(log N) 이하로 유지됨.
복잡도
N번의 Union/Find 연산: O(N · α(N)) ≈ O(N). α는 역 아커만 함수 — 실용적으로 상수.
// 의사 코드 int Find(int x) { if (parent[x] != x) parent[x] = Find(parent[x]); // 경로 압축 return parent[x]; } void Union(int x, int y) { int rx = Find(x), ry = Find(y); if (rx == ry) return; if (rank[rx] < rank[ry]) Swap(ref rx, ref ry); parent[ry] = rx; // 랭크 병합 if (rank[rx] == rank[ry]) rank[rx]++; }
4. SpatialHash — 3D 공간 해시
참조: Pipeline/Utils/SpatialHash.cs
목적
3D 공간을 규칙적인 격자(셀)로 분할해, 특정 점 근처의 객체를 O(1) 근사 시간으로 탐색합니다. NodeEquivalence의 O(N²) 전수 탐색을 O(N)으로 줄이는 핵심 자료구조입니다.
동작 방식
1. 셀 크기(SpatialCellSizeMm = 200mm)로 좌표를 나눠 정수 격자 인덱스 (ix, iy, iz) 계산
2. Dictionary<(ix,iy,iz), List<T>>에 객체 등록
3. 근처 탐색: 쿼리 점의 격자 인덱스 ±1 범위(27개 셀)만 확인
4. 27개 셀 내 객체들과만 거리 계산 → 전체 대비 극소수 후보
셀 크기 선택 기준
셀 크기는 탐지하려는 거리(NodeMergeTol=1mm)의 수십~수백 배여야 합니다.
200mm 셀에서 1mm 노드는 한 셀에 수백 개가 모일 수 있지만, 인접 셀 27개만 확인하면 충분.
셀이 너무 크면 후보가 너무 많아 속도 이점이 사라짐. 너무 작으면 경계 탐지 누락 위험.
// 의사 코드 (int, int, int) CellOf(Point3 p) => ( (int)Math.Floor(p.X / CellSize), (int)Math.Floor(p.Y / CellSize), (int)Math.Floor(p.Z / CellSize) ); void Insert(T item, Point3 pos) { var key = CellOf(pos); cells.GetOrCreate(key).Add(item); } IEnumerable<T> GetNear(Point3 pos) { var (cx, cy, cz) = CellOf(pos); for (int dx=-1; dx<=1; dx++) for (int dy=-1; dy<=1; dy++) for (int dz=-1; dz<=1; dz++) if (cells.TryGetValue((cx+dx,cy+dy,cz+dz), out var list)) foreach (var item in list) yield return item; }
5. U-bolt 2-Phase Snap (UboltRbe)
참조: Pipeline/ElementModifier/UboltSnapToStructureModifier.cs
목적
U-bolt는 파이프를 구조물에 고정하는 클램프입니다. U-bolt의 위치(힌트 좌표)에서 가장 가까운 구조 빔 위 점으로 "snap"하고, 그 점에 RBE2(강체 연결) 요소를 생성합니다.
Phase 1 — 힌트 기반 snap
U-bolt 힌트 좌표(RawPipeRow에서 파싱한 위치)에서 UboltSnapMaxDist(50mm) 이내의 Structure 빔 요소를 SpatialHash로 탐색.
찾은 요소 위에 힌트 좌표를 투영(Segment3.ClosestPointTo)해 snap 위치 결정.
투영점에 새 노드 삽입 → RigidElement(Remark="UBOLT") 생성.
Phase 2 — Fallback snap
Phase 1 실패(힌트 없음 또는 50mm 초과) 시: Pipe 끝점에서 가장 가까운 Structure 노드 탐색.
거리 < UboltSnapMaxDist(50mm)이면 해당 노드를 마스터로 RBE2 생성.
거리 ≥ 50mm이면 snap 실패 → NodeTags.Boundary 태그 + UBOLT_SNAP_FAILED 진단.
Snap 실패 노드는 NodeTags.Boundary로 표시해 FinalValidation에서 자유단 경고를 억제합니다. 그러나 실제 구조 연결이 없으므로 검토 필요.
6. 요소 등분할 메싱 (Meshing)
신규 구현 (HiTess 직접 참조 없음)
목적
FEM 해석 정밀도를 위해 너무 긴 빔 요소를 균등하게 분할합니다. 긴 요소는 굽힘 응력 분포를 부정확하게 표현하므로, 최대 길이 이하로 잘게 나눕니다.
분할 규칙
Structure: 길이 > MeshingMaxLengthStructure(2000mm)이면 분할
Pipe: 길이 > MeshingMaxLengthPipe(1000mm)이면 분할
Equipment: 분할 안 함 (집중 질량은 위치 변경 불가)

분할 개수 = ceil(length / maxLength)
각 조각의 길이 = length / n (균등 분할)
중간 노드 좌표 = 선형 보간으로 계산
단면/재료 유지
분할된 조각들은 원래 요소와 동일한 PropertyId(단면)와 MaterialId(재료)를 가집니다. Orientation(보 방향 벡터)도 동일하게 유지.
// 의사 코드 foreach (var elem in model.Elements.ToList()) { var seg = elem.GetSegment(model); int n = (int)Math.Ceiling(seg.Length / MaxLength(elem.Category)); if (n <= 1) continue; var pts = Enumerable.Range(0, n+1) .Select(i => seg.Start + i * (seg.End - seg.Start) / n) .ToList(); var nodes = pts.Select((p,i) => i == 0 ? elem.StartNodeId : i == n ? elem.EndNodeId : model.AddNode(p).Id).ToList(); model.Remove(elem); for (int i = 0; i < n; i++) model.AddElement(nodes[i], nodes[i+1], elem.PropertyId, elem.Category); }
7. 그룹 연결 (GroupConnect)
참조: Pipeline/ElementModifier/ElementGroupTranslationModifier.cs
목적
CSV 좌표 오차 또는 설계 의도로 인해 물리적으로 연결되어야 하는 요소들이 위상적으로 분리되어 있을 수 있습니다. 이 Stage는 분리된 그룹들을 감지하고 가장 큰 그룹에 연결합니다.
알고리즘 단계
1. UnionFind로 BeamElement 연결 그룹 식별 (공유 노드를 가진 요소들이 같은 그룹)
2. 가장 많은 요소를 포함한 그룹을 "메인 그룹"으로 선정
3. 비메인 그룹의 각 요소에서 메인 그룹의 모든 요소까지 거리 계산 (SpatialHash 활용)
4. 거리 < UboltSnapMaxDist × 2 = 100mm이면 중간 연결 노드 생성 (NodeTags.Boundary 태그)
5. 거리 ≥ 100mm이면 고립 그룹으로 판정 → ISOLATED_GROUP 진단 발행
고립 그룹(100mm 초과)은 자동 연결하지 않습니다. 별도 검토 후 수동으로 처리해야 합니다. FinalValidation에서 자유단 노드 경고로 이어질 수 있습니다.
8. Nastran BDF 8열 고정폭 포맷 (BdfField)
참조: Exporter/BdfFormatFields.cs, BdfBuilder.cs
Nastran BDF 형식 규칙
Nastran BDF(Bulk Data File)는 각 필드가 정확히 8자여야 합니다(고정폭 형식).
1줄 = 80자 = 카드명(8자) + 필드 8개 × 8자 = 72자 + 8자(계속 여부)
실수는 "e" 없이 지수 표기: 2.06e+05 → "2.06+05" (8자 제한 때문에 e 생략)
BdfField 구현
L(int v): 정수를 오른쪽 정렬 8자 문자열로
R(double v): 실수를 8자 내에서 최대 유효 숫자로 (예: "1234.567")
RSci(double v): 지수 표기 8자 — "1.23+05" 형태 (e 없음, Nastran 전용)
RBE2 Continuation 처리
RBE2 슬레이브 노드가 많으면 1줄(80자)에 다 들어가지 않음.
80자 초과 시 "+" 기호로 다음 줄에 이어 씀 (Nastran 규격 필수).
Verify.Xunit 골든 스냅샷으로 포맷 회귀를 방지.
// 예시 GRID 카드 (노드 42, 좌표 100.0, 200.5, 0.0) // "GRID "+" 42"+" "+" 100.000"+" 200.500"+" .0" GRID 42 100.000 200.500 .0 // 예시 RBE2 카드 (마스터 10, 슬레이브 11 12 13 14 15 16 17 ...) RBE2* 1* 10*123456* 11 +* 12* 13* 14* 15 +* 16* 17
